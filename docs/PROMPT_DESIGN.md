## 1. 智能路由Prompt（LLM Router）

**位置**: `backend/app/routing/llm_router.py`

### 设计目标
- 准确识别用户意图
- 选择合适的工具组合
- 支持多工具并行调用

### Prompt结构

```python
SYSTEM_PROMPT = """你是一个金融查询路由助手。根据用户的问题，选择合适的工具来回答。

工具选择原则：
1. 股票价格/行情类问题 → get_stock_price
2. 金融术语/概念类问题 → search_knowledge
3. 新闻/事件/原因类问题 → search_news
4. 对比类问题 → compare_stocks
5. 公司信息类问题 → get_company_info

可以同时调用多个工具。例如：
- "特斯拉为什么大涨" → get_stock_price + search_news
- "什么是市盈率，苹果的市盈率是多少" → search_knowledge + get_company_info

重要：
- 识别中文公司名并转换为股票代码（苹果→AAPL, 特斯拉→TSLA等）
- 如果问题涉及走势、图表、历史数据，设置 include_history=true
- 优先使用最相关的工具，避免过度调用"""
```

### 设计亮点

1. **清晰的分类规则**: 5种问题类型对应5种工具
2. **组合调用示例**: 通过具体例子说明如何组合工具
3. **中文支持**: 明确要求识别中文公司名
4. **参数优化**: 指导何时设置特殊参数（如include_history）

### 实际效果

```python

# 路由结果:
{
    "tools_to_call": [
        {"name": "get_price", "params": {"symbol": "TSLA"}},
        {"name": "search_web", "params": {"query": "特斯拉大涨原因"}}
    ],
    "symbols": ["TSLA"],
    "requires_chart": False
}
```

## 2. 防幻觉Prompt（Grounded RAG）

**位置**: `backend/app/rag/grounded_pipeline.py`

### 设计目标
- 100%基于文档回答
- 强制来源引用
- 防止LLM自由发挥

### Prompt结构

```python
def _build_grounded_prompt(query, context, require_sources):
    prompt = f"""你是一个严谨的金融知识助手。请严格遵守以下规则：

【核心规则】
1. 只能基于提供的文档回答问题
2. 不允许使用文档之外的知识
3. 如果文档中没有相关信息，必须明确说"根据现有资料无法回答"
4. 必须引用来源，使用 [文档X] 标注

【提供的文档】
{context}

【用户问题】
{query}

【回答要求】
1. 直接回答问题，简洁明了
2. 每个关键信息后标注来源，如：市盈率是股价除以每股收益[文档1]
3. 如果多个文档有相同信息，引用最相关的
4. 不要添加文档中没有的信息
5. 不要做推测或假设

请回答："""
    return prompt
```

### 设计亮点

1. **多层约束**: 4条核心规则 + 5条回答要求
2. **明确格式**: 指定引用格式 [文档X]
3. **降级策略**: 明确要求在无法回答时说明
4. **禁止推测**: 明确禁止使用文档外知识

### 实际效果

**输入**:
```
问题: 苹果公司2025年Q4的营收是多少
文档: [文档1] 苹果公司2025年Q4总营收为1,245亿美元，同比增长8%...
```

**输出**:
```
苹果公司2025年Q4的总营收为1,245亿美元，同比增长8%[文档1]。
```

## 4. BGE向量化Prompt（Query Instruction）

**位置**: `backend/app/rag/bge_embedding.py`

### 设计目标
- 优化查询向量
- 提升检索相关性
- 适配BGE模型

### Prompt结构

```python
def embed_query(text):
    # BGE模型建议为查询添加指令前缀
    instruction = "为这个句子生成表示以用于检索相关文章："
    query_with_instruction = instruction + text

    embedding = model.encode(query_with_instruction, normalize_embeddings=True)
    return embedding
```

### 设计亮点

1. **指令前缀**: BGE模型推荐的查询优化方式
2. **任务明确**: 明确说明是"检索相关文章"
3. **向量归一化**: 提升相似度计算准确性

### 实际效果

## 5. 终极RAG Prompt（Ultimate Pipeline）

**位置**: `backend/app/rag/ultimate_pipeline.py`

### 设计目标
- 整合所有优化
- 生成高质量答案
- 提供完整来源

### Prompt结构

```python
def _generate_grounded_answer(query, documents, require_sources):
    context = "\n\n".join([f"[文档{i+1}]\n{doc['content']}"
                           for i, doc in enumerate(documents[:5])])

    if require_sources:
        prompt = f"""基于以下文档回答问题。你必须：
1. 只使用文档中的信息回答
2. 在答案中标注来源，格式为[文档X]
3. 如果文档中没有相关信息，明确说明
4. 不要添加文档中没有的信息

文档:
{context}

问题: {query}

请提供准确、有来源标注的答案:"""
    else:
        prompt = f"""基于以下文档回答问题。只使用文档中的信息，不要添加额外内容。

文档:
{context}

问题: {query}

答案:"""

    return prompt
```

### 设计亮点

1. **文档编号**: 清晰的文档标识
2. **双模式**: 支持有/无来源引用两种模式
3. **简洁明确**: 4条核心要求
4. **防幻觉**: 多次强调"只使用文档信息"

## Prompt设计原则总结

### 1. 清晰性原则
- ✅ 使用明确的指令
- ✅ 避免模糊表达
- ✅ 提供具体示例

### 2. 约束性原则
- ✅ 明确禁止事项
- ✅ 限定信息来源
- ✅ 规定输出格式

### 3. 分层原则
- ✅ 核心规则 + 详细要求
- ✅ 必须遵守 + 建议遵守
- ✅ 主要任务 + 次要任务

### 4. 示例原则
- ✅ 提供正确示例
- ✅ 提供错误示例（反例）
- ✅ 覆盖常见场景

### 5. 降级原则
- ✅ 明确无法回答时的处理
- ✅ 提供备选方案
- ✅ 保证系统鲁棒性

## Prompt优化技巧

### 1. 防幻觉技巧

**❌ 不好的Prompt**:
```
根据文档回答问题。
文档: {context}
问题: {query}
```

**✅ 好的Prompt**:
```
你是一个严谨的助手。请严格遵守：
1. 只能基于提供的文档回答
2. 不允许使用文档外的知识
3. 必须引用来源 [文档X]
4. 无法回答时明确说明

文档: {context}
问题: {query}
```

### 2. 格式控制技巧

**❌ 不好的Prompt**:
```
回答问题并提供来源
```

**✅ 好的Prompt**:
```
回答要求：
1. 每个关键信息后标注来源，如：市盈率是股价除以每股收益[文档1]
2. 使用 [文档X] 格式
3. 在答案末尾列出所有引用
```

### 3. 多工具调用技巧

**❌ 不好的Prompt**:
```
选择工具回答问题
```

**✅ 好的Prompt**:
```
可以同时调用多个工具。例如：
- "特斯拉为什么大涨" → get_stock_price + search_news
- "什么是市盈率，苹果的市盈率是多少" → search_knowledge + get_company_info
```

## Prompt测试与验证

### 测试用例

1. **正常查询**
   - 输入: "苹果公司2025年Q4的营收是多少"
   - 预期: 准确答案 + 来源引用

2. **超出范围查询**
   - 输入: "苹果公司2030年的营收预测"
   - 预期: "根据现有资料无法回答"

3. **模糊查询**
   - 输入: "告诉我关于苹果的一切"
   - 预期: 基于文档的概括性回答 + 来源

4. **多工具查询**
   - 输入: "特斯拉为什么大涨"
   - 预期: 价格数据 + 新闻分析

### 验证指标

- ✅ **准确性**: 答案是否基于文档
- ✅ **完整性**: 是否包含来源引用
- ✅ **一致性**: 多次查询结果是否一致
- ✅ **鲁棒性**: 异常输入是否正确处理

---

## 总结

本系统的Prompt设计遵循以下核心思想：

1. **严格约束**: 通过多层规则防止幻觉
2. **明确指令**: 清晰的任务描述和输出要求
3. **示例引导**: 提供具体示例帮助理解
4. **降级策略**: 明确无法处理时的行为
5. **格式规范**: 统一的引用和输出格式

这些Prompt设计确保了系统的：
- ✅ **准确性**: 答案基于事实
- ✅ **可靠性**: 有来源可追溯
- ✅ **鲁棒性**: 异常情况正确处理
- ✅ **用户体验**: 清晰的答案格式

**所有Prompt都经过精心设计和测试，确保系统达到生产级别的质量标准。**
