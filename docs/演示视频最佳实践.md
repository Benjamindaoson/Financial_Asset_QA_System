# 演示视频最佳实践指南

## 🎯 演示目标

确保在60秒内展示系统的**核心价值**和**技术亮点**，让评审者快速理解系统能力。

---

## 📹 Part 1: 资产问答示例（60秒）

### 演示策略：展示"数据驱动 + AI分析"的完整链路

#### 推荐演示流程（按顺序）

**1. 简单查询 → 复杂分析（15秒）**
```
问题1: "特斯拉现在多少钱？"
目的: 展示实时数据获取速度
亮点:
- 秒级响应
- 数据源标注（yfinance/Finnhub）
- 时间戳显示
```

**2. 涨跌分析 + 图表（20秒）**
```
问题2: "特斯拉最近30天涨跌如何？"
目的: 展示技术分析能力
亮点:
- 自动生成K线图
- 涨跌幅计算（+12.5%）
- 波动率、最大回撤等指标
- 趋势判断（上涨/震荡）
```

**3. 新闻事件分析（25秒）**
```
问题3: "特斯拉为什么最近大涨？"
目的: 展示AI推理能力
亮点:
- 自动调用Web Search
- 结合实时新闻
- 结构化分析（财报/政策/市场情绪）
- 数据来源标注
```

### 演示技巧

#### ✅ 做什么
1. **提前准备好问题** - 复制粘贴，避免打字错误
2. **选择热门股票** - TSLA/AAPL/NVDA（数据更新快）
3. **展示流式响应** - 让评审看到AI"思考"过程
4. **突出数据源** - 指出"数据来自Finnhub"等标注
5. **展示图表** - 鼠标悬停在图表上显示具体数值

#### ❌ 避免什么
1. **不要选冷门股票** - 可能数据缺失
2. **不要问预测问题** - "明天会涨吗"系统会拒绝
3. **不要等太久** - 如果超过5秒无响应，切换问题
4. **不要展示错误** - 提前测试确保API正常

### 最佳演示脚本

```markdown
【画外音】
"首先演示资产问答功能。我问：特斯拉现在多少钱？"

【操作】输入问题，等待响应

【画外音】
"可以看到，系统在1秒内返回了实时价格$285.32，
数据来自Finnhub API，并标注了时间戳。"

【操作】输入第二个问题

【画外音】
"接下来问：特斯拉最近30天涨跌如何？
系统自动生成了K线图，显示30天上涨了12.5%，
并计算了波动率、最大回撤等技术指标。"

【操作】输入第三个问题

【画外音】
"最后问：特斯拉为什么最近大涨？
系统调用了Web Search，结合最新新闻，
分析出三个主要原因：Q4财报超预期、
Cybertruck交付量增长、以及AI自动驾驶进展。
注意，所有分析都有数据来源标注。"
```

4. **不要展示低置信度** - 提前测试确保高置信度

## 💡 加分技巧

### 1. 对比展示
在演示RAG时，可以说：
> "如果没有RAG，AI可能会编造数据。
> 但我们的系统从知识库检索，确保答案准确。"

### 2. 突出技术亮点
- **流式响应**: "注意AI是逐字输出的，这是流式响应技术"
- **数据源标注**: "每个数据都标注了来源，确保可追溯"
- **置信度评分**: "系统会评估答案置信度，低于阈值会拒绝回答"
- **智能路由**: "系统自动判断问题类型，调用不同工具"

### 3. 展示容错能力
可以故意问一个超纲问题：
```
"比特币明天会涨吗？"
→ 系统拒绝："我不能预测未来走势"
```
然后说：
> "系统会拒绝回答预测类问题，避免误导用户。"

### 4. 展示多工具组合
问一个复杂问题：
```
"特斯拉现在多少钱？为什么涨了？"
→ 系统同时调用价格API + Web Search
```
然后说：
> "系统智能组合了多个工具，一次性回答了两个问题。"

## 📊 演示效果评估标准

### 优秀演示应该展示：
1. ✅ **速度快** - 响应时间<3秒
2. ✅ **数据准** - 所有数据有来源标注
3. ✅ **结构清晰** - 表格/列表/图表
4. ✅ **智能路由** - 自动选择工具
5. ✅ **防幻觉** - 置信度评分/来源标注
6. ✅ **用户体验** - 流式响应/实时图表

### 避免的问题：
1. ❌ 响应超时
2. ❌ 数据错误
3. ❌ 回答模糊
4. ❌ 没有来源
5. ❌ 界面卡顿
